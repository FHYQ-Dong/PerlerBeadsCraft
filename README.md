# PerlerBeadsCraft

把任意图片转换成**拼豆（Perler / Hama beads）图纸**。

输入一张图片，输出三样东西：

| 产物             | 文件                           | 用途                                                                                    |
| ---------------- | ------------------------------ | --------------------------------------------------------------------------------------- |
| 彩色成品预览     | `<名字>_preview.png`         | 模拟拼好后的样子（圆珠 + 拼板网格）                                                     |
| 打印图纸         | `<名字>_chart.png`           | 每格直接标注色号、坐标轴、每 5 格虚线 / 每 10 格加粗、右侧色号图例                      |
| 用料清单         | `<名字>_beads.csv`           | 每种颜色的色号、名称、HEX、所需颗数及合计                                               |
| 分色地图（可选） | `<名字>_colormaps/` 子文件夹 | 每种颜色一张图，只标该色珠子位置、其余留白，方便一次只摆一种颜色（加 `--color-maps`） |

内置两套色卡：

- **`mard`（默认）** — MARD（漫的）221 标准色，色号如 `A5` `B8` `F8`。
- **`hama`** — Hama Midi 53 标准色，色号如 `H18`。

用 `-p hama` 切换。颜色匹配在 **CIE Lab** 色彩空间做最近邻，比 RGB 欧氏距离更贴近人眼感受。透明像素（alpha 低于阈值）会被识别为「空位」，不放珠子。

## 环境

- Python ≥ 3.14，使用 [uv](https://docs.astral.sh/uv/) 管理。
- 依赖：`pillow`、`numpy`、`matplotlib`（已写入 `pyproject.toml`）。

首次准备环境：

```bash
uv sync
```

## 使用

```bash
# 基本用法：宽 48 颗珠子，保持宽高比
uv run --no-sync perler 图片.png

# 指定宽度、限制用色数、自定义输出目录
uv run --no-sync perler 图片.png -w 58 -m 16 -o 我的图纸

# 也可以不依赖 console 脚本，直接用模块运行
uv run --no-sync python -m perlerbeadscraft 图片.png -w 40
```

环境装好后，`--no-sync` 仅用于规避上面提到的镜像源问题；网络正常时可省略。

### 参数

| 参数                     | 说明                                                                                                   | 默认              |
| ------------------------ | ------------------------------------------------------------------------------------------------------ | ----------------- |
| `image`                | 源图片（png/jpg/...）                                                                                  | 必填              |
| `-w, --width`          | 图纸宽度（珠子数）                                                                                     | 不指定尺寸时为 48 |
| `--height`             | 图纸高度（珠子数）。只给宽或高之一即按比例缩放                                                         | —                |
| `-m, --max-colors`     | 限制用色数：用 k-means 聚类选出 N 个有代表性的颜色（保留关键点缀色，不偏向大面积主色系）               | 不限制            |
| `--protect-accents`    | 配合 `-m`：保护「面积小但色相独特」的点缀色不被合并掉（详见下文）                                    | 关                |
| `--min-color-distance` | 配合 `-m`：所选颜色两两间的最小 Lab 距离（deltaE），避免「好几个几乎一样的浅色」浪费名额。建议 8–12 | 0（关）           |
| `--saturation`         | 转换前提高(>1)/降低(<1)饱和度。粉彩/发灰的图建议 1.4–1.6                                              | 1.0               |
| `--contrast`           | 转换前提高(>1)/降低(<1)对比度。建议 1.1 左右                                                           | 1.0               |
| `-p, --palette`        | 色卡：`mard`（221 标准色）/ `hama`（53 色）                                                        | `mard`          |
| `-o, --out-dir`        | 输出目录                                                                                               | `output`        |
| `--bead-size`          | 预览图中每颗珠子的像素大小                                                                             | 24                |
| `--shape`              | 预览珠子形状：`circle` / `square`                                                                  | `circle`        |
| `--alpha-threshold`    | 源像素 alpha 低于此值视为空位（0–255）                                                                | 128               |
| `--no-codes`           | 图纸/分色地图不在每格标注色号                                                                          | 关                |
| `--color-maps`         | 额外生成 `<名字>_colormaps/` 子文件夹，每色一张分色地图                                              | 关                |
| `--no-grid`            | 预览图不画网格线                                                                                       | 关                |

## 高调/粉彩图技巧

柔和的粉彩插画（背景、皮肤、裙子都很浅）容易转出来「发灰糊成一片、五官丢失」，因为浅色占面积大，名额都花在几乎一样的浅色上。组合用这三个参数能明显改善：

```bash
uv run --no-sync perler 图片.png -w 74 -m 16 \
  --saturation 1.5 --contrast 1.12 --min-color-distance 10 --protect-accents
```

- `--saturation/--contrast`：把挤在一起的浅色拉开，眼睛、轮廓回来（效果最明显的一步）。
- `--min-color-distance 10`：禁止选两个过于接近的颜色，把「5 个一样的浅粉」换成更有变化的配色。
- `--protect-accents`：保住眼神光这种小点缀。

## 工作原理

1. **预调整**（可选）：按 `--saturation`/`--contrast` 调整图像——只影响选色与匹配，珠子色块仍按真实豆色显示（`pattern.py`）。
2. **缩放**：把图片用 LANCZOS 缩到「珠子网格」大小，按比例保持宽高。
3. **匹配**：每格 RGB 转 Lab，与色卡每个颜色算 Lab 距离取最近色（`color.py`）。
4. **减色**（可选，`-m N`）：在 Lab 空间对所有格子的颜色做 **k-means 聚类**，得到 N 个有代表性的色团，把每个色团中心吸附到最近的豆色，组成调色板子集；再让每格在子集里就近取色（`pattern.py`）。
   - 这样选出的是**铺开、互相区分**的颜色，而不是「出现次数最多的近似色」。面积小但关键的点缀色（蓝眼睛、红嘴唇）会自成一团被保住，不会被大面积的主色系吞掉。
   - 结果稳定可复现（k-means 固定随机种子）。
   - **最小色距**（`--min-color-distance`）：用更大的候选池聚类，再按面积贪心、跳过与已选色过近的，保证所选颜色彼此区分，不浪费名额在近似色上。
   - **小点缀保护**（`--protect-accents`）：极小的点缀色（如眼神光、小蝴蝶结）有时仍会被 k-means 漏掉。开启后会找出「被现有配色表示得很差」的格子聚成团，若某团够大且色相独特，就把它**换进配色**——代价是丢掉子集里一对**近似重复色**中较少用的那个。只有「真的漏了点缀」且「有近似色可让」时才动手，所以不会把已经选好的配色搞坏；没有可让的近似色时不动（这时该调大 `-m`）。
5. **渲染**：
   - 预览图用 Pillow 把每格画成带孔圆珠 + 拼板网格（`render.py`）。
   - 图纸用 matplotlib 在每格标注色号 + 坐标 + 图例。

## 代码结构

```
perlerbeadscraft/
  palettes.py   # 色卡注册（mard / hama）+ BeadColor / Palette 数据结构
  _mard_data.py # MARD 221 标准色数据（含色号 + HEX）
  color.py      # sRGB→Lab 转换、最近邻匹配、Lab k-means（numpy 向量化）
  pattern.py    # 图片→珠子网格、聚类减色、用量统计
  render.py     # 彩色预览 + 色号图纸 + 图例 + 分色地图
  cli.py        # 命令行入口（argparse）
main.py         # 等价入口：uv run main.py ...
```
